# BeerGuy Monitor

Modular Telegram bot for monitoring BeerGuy / BGUY on Solana.

## Structure

- `main.py` wires the Telegram application.
- `config.py` loads `.env` values.
- `bot/` contains commands, keyboards, callbacks, and alert sending.
- `chain/` contains Solana RPC, Helius, Birdeye, DexScreener, pool config, and transaction classification helpers.
- `services/` contains price, liquidity, holders, volume, LP rewards, fixed replies, community replies, and status services.
- `storage/` contains JSON and asset helpers.
- `assets/images/` contains alert/reply images. Missing images are skipped safely.

## Community bot features

BeerGuy Monitor can also act as a community bot when the community options are enabled.

### Automatic greetings

Set `ENABLE_AUTO_REPLIES=true` to let the bot answer common greetings without AI. The bot uses preset randomized replies and optional images from `assets/images/`:

- `hi`, `hello`, `hey`, `salut`, `bună`, `buna`, `bună ziua`, `buna ziua` use `greeting.png` or `welcome.png`.
- `gm`, `good morning`, `bună dimineața`, `buna dimineata` use `good_morning.png`.
- `good afternoon`, `good evening`, and `noapte bună` use category-specific text replies.

If an image is missing, the bot safely sends only text.

### Welcome and goodbye messages

Set `ENABLE_WELCOME_MESSAGES=true` to greet new members with `welcome.png` and a BeerGuy welcome message. Set `ENABLE_GOODBYE_MESSAGES=true` to send the optional goodbye message when a member leaves.

### Keyword reactions

When automatic replies are enabled, the bot occasionally reacts to community keywords: `beer`, `cheers`, `raid`, `brew`, `farm`, `bguy`, and `beerguy`. These reactions are intentionally random so the bot does not answer every mention.

### Anti-spam behavior

`AUTO_REPLY_COOLDOWN=60` limits automatic community replies to at most one message per 60 seconds. The bot also avoids replying to the same user twice in a row, waits longer before responding to the same user again, ignores Telegram commands, and ignores messages generated by other bots.

## Fixed replies

The fixed reply service still handles deterministic utility replies such as `buy`, `chart`, and `price`. Community greetings are handled by `services/community.py`.

### Weekly Activity Leaderboard

Set `LEADERBOARD_ENABLED=true` to score real Telegram community engagement. Scoring is intentionally quiet so normal chat remains natural: normal messages earn 1 point, replies earn 2 extra points, useful BeerGuy-related discussion earns 3 extra points, media/memes/GIFs earn 2 points, and detectable member invites earn 5 points. Bot messages, spam, repeated messages, scam links, deleted/moderated messages, and low-effort short messages are not rewarded.

Anti-abuse defaults limit each user to one scored message every 60 seconds and 30 points per day. Points are persisted in `data/leaderboard.json` so Railway restarts do not clear the current week.

```env
LEADERBOARD_ENABLED=true
LEADERBOARD_POST_DAY=saturday
LEADERBOARD_POST_HOUR=18
LEADERBOARD_TIMEZONE=Europe/Berlin
LEADERBOARD_MAX_DAILY_POINTS=30
LEADERBOARD_MESSAGE_COOLDOWN_SECONDS=60
```

Commands:

- `/leaderboard` shows the current weekly top 10 Beer Raiders.
- `/leaderboard_reset` resets the current week (admin only).
- `/leaderboard_export` exports the JSON store (admin only).
- `/leaderboard_pause` pauses scoring (admin only).
- `/leaderboard_resume` resumes scoring (admin only).

The bot automatically posts the weekly leaderboard to `TELEGRAM_GROUP_CHAT_ID` on the configured day/hour, then resets weekly points for the next week.
